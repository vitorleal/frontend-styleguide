Objetivo deste Guia

Escrever códigos é o que fazemos todos os dias. Porém muitos desses códigos não vão passar apenas por uma única pessoa, mas sim por muitas. Por isso criamos esse guia, para que você possa manter um padrão de convenções e boas práticas para que seu código seja o mais declarativo possível e que ele possa ter um brilho natural.

Toda linha de código deve parecer que foi escrita por uma única pessoa, não importa a quantidade de contribuidores.

Siga até sua morte!

  • Readability counts.
  • Seja consistente.
  • Cada objeto deve ter apenas uma responsabilidade.
  • SOLID para qualquer coisa.
  • Seja sempre DRY (Don't Repeat Yourself).
  • Convenção sobre configuração (CoC).
  • Flat é melhor que aninhado
  • Bonito é melhor que feio.
  • Simples é melhor que complexo.
  • Adicione uma linha em branco no final de todo arquivo.
  • Limite suas linhas à 80 caracteres.

Referências para o dia-dia

Um padrão é algo determinado por uma ou por algumas pessoas. Não existe o melhor ou o pior padrão, existe aquele que você mais se adequar. Por isso, caso você não se interesse por este guia de estilo, ou mesmo que se interesse, porém queira conhecer outros, preparamos uma lista bem interessante para você. Fiquei a vontade para estudar e montar o seu próprio.
